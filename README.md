# 苏州高中目标查询助手（微信小程序 MP2）

这是“苏州高中目标查询助手”的原生微信小程序项目，当前版本为 `1.1.0`，状态为“上架准备版”。项目使用 WXML、WXSS、JavaScript 和 JSON，不依赖第三方 UI 库，不新增 npm 依赖。

本仓库只放微信小程序代码。Flutter iOS/App 软件项目保留在 `/Users/tom/Dev/suzhou_highschool_app`，两个项目不要混在同一个仓库或同一个微信开发者工具项目中。

## 当前版本能做什么

- 查看苏州高中阶段学校基础信息。
- 按区域、学校类型、办学性质、住宿状态和数据状态筛选。
- 搜索学校名称、区域、类型、性质和标签。
- 收藏关注学校。
- 记录阶段学习目标，自动计算与本阶段目标的差距。
- 查看数据来源、隐私说明和提交前说明。

## 当前版本不做什么

- 不展示真实录取分数线。
- 不做录取预测。
- 不提供志愿填报结论。
- 不请求定位，不内置地图 SDK。
- 不接后台、云开发、数据库、登录、支付、广告、推送或第三方统计 SDK。
- 不上传收藏、目标记录或草稿。

## 页面列表

- 首页：产品定位、功能边界、快捷入口、使用步骤。
- 学校库：搜索、筛选、收藏、进入详情、空状态。
- 学校详情：基础信息、来源信息、信息完整度、复制操作。
- 收藏：查看收藏、取消收藏、空状态跳转。
- 目标记录：阶段测评分数、阶段目标分数、备注、记录管理。
- 数据说明：数据范围、来源、暂不展示内容和更新计划。
- 隐私说明：本地缓存、不登录、不上传、不定位等说明。
- 我的：版本状态、收藏数量、目标记录数量、清除本地数据。

## 学校数据来源说明

学校库当前包含 35 条数据，其中真实公开来源样本 33 条、示例学校 2 条。真实学校优先使用学校官网、教育局官网、政府公开网站或公开教育信息平台；无法完全核实的字段用“待核实”或来源说明标注。

每条学校数据包含 `sourceTitle`、`sourceUrl`、`sourceCheckedAt`、`sourceNote` 和 `dataStatus`。当前版本不把“尽量核对”表述为“绝对准确”，最终仍以学校、教育局和招生主管部门发布为准。

详见 [MP2 学校数据来源](docs/mp2_school_data_sources.md)。

## 隐私说明

本版本无需登录。收藏、目标记录和输入草稿只保存在微信小程序本地缓存；不上传服务器，不获取定位，不接入支付、广告、推送或第三方统计 SDK。备注框会提醒用户不要填写姓名、手机号、身份证、准考证号等个人敏感信息。

用户可在“我的”页面清除收藏、目标记录和草稿。详见 [MP2 隐私声明](docs/mp2_privacy_statement.md)。

## 导入微信开发者工具

1. 打开微信开发者工具，选择“导入项目”。
2. 项目目录选择本目录：`/Users/tom/Dev/suzhou_highschool_miniprogram`。
3. 本地预览可保留 `project.config.json` 中的 `touristappid`。
4. 上传开发版本前，在微信开发者工具或 `project.config.json` 中绑定自己的真实小程序 AppID。
5. 不要把 AppSecret、API Key、Token、Cookie、账号密码写入代码、文档或 Git。

## touristappid 说明

`project.config.json` 当前保持：

```json
"appid": "touristappid"
```

这只用于本地导入和预览。上传体验版、提交审核、发布前，必须替换为微信公众平台的小程序真实 AppID。AppSecret 不应出现在本仓库。

## 本地验证命令

```sh
node scripts/verify_mp1.js
node scripts/verify_mp2.js
node scripts/smoke_local_logic.js
node scripts/smoke_page_logic.js
find . -type f -name '*.js' -not -path './.git/*' -print0 | xargs -0 -n1 node --check
```

这些脚本只提供本地静态和逻辑补充检查，不代表微信开发者工具编译通过。提交审核前仍必须在微信开发者工具中完成编译、页面操作和真机预览。

## 上架前人工步骤

- 注册或确认小程序账号。
- 完成主体认证和小程序备案。
- 选择合适服务类目。
- 替换真实 AppID。
- 在微信公众平台填写隐私保护指引。
- 在微信开发者工具中真机预览。
- 上传体验版并复测。
- 填写版本说明和审核备注。
- 提交审核。
- 审核通过后发布。

详见 [MP2 提交前清单](docs/mp2_submission_checklist.md) 和 [MP2 审核材料](docs/mp2_review_materials.md)。
