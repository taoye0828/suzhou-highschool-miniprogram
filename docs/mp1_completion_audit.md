# MP1 完成审计

审计日期：2026-07-02

## 范围与结果

- 独立原生微信小程序项目已创建，可由微信开发者工具按目录导入。
- 已配置首页、学校库、学校详情、收藏、目标记录、数据说明、隐私说明、我的八类页面。
- 首页、学校库、收藏、目标记录、我的已加入 tabBar。
- 学校搜索及区域、类型、住宿筛选已实现，并提供空状态。
- 收藏的新增、取消、列表查看和详情跳转已实现。
- 目标记录的分数输入、备注、差距计算、保存、删除、清空输入和清空全部已实现。
- 收藏、目标记录和草稿统一使用 `mp1.*` 本地存储键。
- 本地存储读取会校验记录结构，损坏或旧格式异常记录不会使页面崩溃。
- 存储写入、删除和清空失败会返回明确结果，页面不会在失败时提示成功。
- 目标记录最多保存 100 条，草稿采用 400ms 防抖写入。
- 复制学校名称、复制地图搜索词、外部地图搜索说明已实现。
- 未实现直接唤起外部地图：无坐标、无地图 SDK、无定位权限时直接唤起不稳定，当前采用复制搜索词后由用户自行打开地图应用的合规方案。

## 数据边界

- 使用三个明显虚构的演示学校。
- 从 Flutter P4 已核验资产复制三个公开来源学校的安全基础字段。
- 未复制分数、电话、照片、地址、坐标、用户距离或路线信息。
- 正式页面不展示任何真实录取分数数据，也不进行录取判断。

## 外部能力边界

- 无后台、无云开发、无 Supabase、无远程 API。
- 无账号登录、支付、广告、推送、用户上传或第三方统计 SDK。
- 无定位权限、无地图 SDK、无定位或地图位置调用。
- 无 AppSecret、API key、服务端密钥或环境配置文件。

## 验证方法

运行：

```sh
node scripts/verify_mp1.js
node scripts/smoke_local_logic.js
node scripts/smoke_page_logic.js
find . -type f -name '*.js' -not -path './.git/*' -print0 | xargs -0 -n1 node --check
```

静态脚本还会检查 WXML 事件处理器是否存在，并扫描全部项目文本文件中的高风险密钥形态。它仍不能替代微信编译器。

微信开发者工具实际编译、真机预览、上传开发版本和审核仍需用户使用自己的账号与 AppID 手工完成。
