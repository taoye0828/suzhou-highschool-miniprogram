# MP1 微信小程序提交审核前清单

## 代码准备状态

- [x] 原生微信小程序标准目录已建立。
- [x] `app.json` 已注册全部页面并配置五项 tabBar。
- [x] `project.config.json` 可用于微信开发者工具导入。
- [x] 当前 AppID 为本地预览占位值 `touristappid`。
- [x] 当前代码不包含登录、支付、广告、定位、AI 或录取预测功能。
- [x] 当前代码不展示真实录取分数数据。
- [x] 当前代码不连接后台、云开发、Supabase 或远程 API。
- [x] 本地存储、隐私说明、数据说明和清除功能已完成。
- [x] 损坏缓存、存储容量失败和删除失败已通过本地故障注入测试。
- [x] 静态脚本明确标记为补充检查，不再声称替代微信开发者工具编译。

## 用户必须人工完成

- [ ] 注册并认证微信小程序账号。
- [ ] 从微信公众平台获取真实 AppID，并在微信开发者工具中绑定项目。
- [ ] 按微信后台要求完成小程序备案。
- [ ] 选择服务类目；可优先评估“工具”或“教育信息服务”相关方向，最终以后台当时可选类目与资质要求为准。
- [ ] 如平台要求，提交相应主体或行业资质。
- [ ] 确认小程序名称、头像和简介。
- [ ] 在微信开发者工具中完成真机预览和基础功能复测。
- [ ] 确认微信开发者工具实际编译成功；本地静态脚本通过不能替代本项。
- [ ] 使用微信开发者工具上传代码。
- [ ] 在微信公众平台设置体验版并按审核路径复测。
- [ ] 填写版本说明、服务内容、隐私保护说明和审核备注。
- [ ] 提交微信审核。
- [ ] 审核通过后由用户人工点击发布。

## 平台流程说明

代码可导入不等于可直接上架。真实 AppID、小程序账号、备案、类目与可能需要的资质均由用户在微信公众平台人工完成。上传代码后通常需要先设为体验版验证，再提交审核；只有审核通过后才能发布。

## 上传前最终复核

- [ ] 将占位 AppID 替换为用户自己的真实 AppID。
- [ ] 确认项目内没有 AppSecret、密钥、真实用户数据或临时文件。
- [ ] 确认开发者工具无编译错误、页面无明显空白。
- [ ] 逐页测试首页、筛选、详情、收藏、目标记录、清除数据和说明页。
- [ ] 确认隐私保护指引与实际代码行为一致。
- [ ] 确认审核材料没有夸大功能，没有声称已审核或已发布。
