## 如何打开微信开发者工具

可以在 Finder 中打开：

```text
/Applications/wechatwebdevtools.app
```

也可以在终端执行：

```bash
open -a wechatwebdevtools
```

## 如何开启服务端口

当前 CLI 返回“工具的服务端口已关闭”。用户需要在图形界面手动开启：

1. 打开微信开发者工具。
2. 进入“设置”或“偏好设置”。
3. 进入“安全”或“安全设置”。
4. 开启“服务端口”。
5. 开启后可以再次运行 MP9，让 Codex 继续尝试 CLI 打开项目或上传前检查。

Codex 本轮没有自动开启服务端口，也没有绕过确认。

## 如何导入项目

1. 打开微信开发者工具。
2. 选择“小程序”。
3. 选择“导入项目”。
4. 项目目录填写：

```text
/Users/tom/Dev/suzhou_highschool_miniprogram
```

5. AppID 填写你的真实小程序 AppID。
6. 后端服务选择“不使用云服务”。

不要把真实 AppID、AppSecret、token、cookie、账号密码或其他密钥写入仓库文件、README、日志或截图说明。

## 如何编译和手机预览

1. 导入项目后点击“编译”。
2. 观察 Console，确认没有红色报错。
3. 点击“预览”生成二维码。
4. 使用绑定为开发者或体验成员的微信扫码。
5. 在手机端检查首页、学校库、学校详情、收藏、学习目标、数据说明、隐私说明和我的页。

## 如何上传开发版本

编译和手机预览都通过后，在微信开发者工具点击“上传”。

- 版本号：`1.4.0`
- 版本描述：

## 如何设为体验版和添加体验成员

1. 进入微信公众平台。
2. 打开小程序版本管理。
3. 找到刚上传的开发版本。
4. 设置为体验版。
5. 添加体验成员微信号。
6. 让体验成员扫码或从体验入口打开复测。

## 如何提交审核

提交审核前确认：

- AppID 已换成真实 AppID
- 编译通过
- 手机预览通过
- Console 无红色报错
- 不登录
- 不定位
- 不支付
- 不接广告
- 不使用云服务
- 不上传用户个人数据
- 不提供录取预测或志愿填报结论

确认后在微信公众平台提交审核。审核结果以微信平台为准，不能承诺通过。

## Console 报错处理

如果 Console 有红色报错，请复制红色报错全文，并同时记录：

- 当前页面路径
- 触发步骤

- macOS 版本

不要只提供截图，也不要只提供最后一行错误。
