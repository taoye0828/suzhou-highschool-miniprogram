# MP2 上架准备态说明

## 当前版本完成了什么

- 原生微信小程序页面完整：首页、学校库、学校详情、收藏、目标记录、数据说明、隐私说明、我的。
- 学校基础库扩展到 35 条，其中真实公开来源样本 33 条、示例学校 2 条。
- 学校数据补充来源标题、来源链接、查询日期、数据状态和来源说明。
- 学校库支持关键词、区域、学校类型、办学性质、住宿状态和数据状态筛选。
- 学校详情页展示基础信息、来源信息、信息完整度和复制操作。
- 目标记录页定位为“学习目标记录”，不做录取判断。
- 收藏、目标记录、草稿均保存在本机。
- README、审核材料、隐私声明、学校来源清单、提交前清单已补齐。
- 本地验证脚本包含 MP1 静态检查、MP2 静态检查、存储逻辑 smoke、页面逻辑 smoke。

## 当前版本不包含什么

- 不接后台、云开发、数据库或远程服务。
- 不接 AI、登录、支付、广告、推送或第三方统计 SDK。
- 不请求定位，不调用地图 SDK，不展示未经核实坐标。
- 不展示真实录取分数线。
- 不做录取预测，不提供志愿填报结论。
- 不收集姓名、手机号、身份证、准考证号等个人敏感信息。

## 为什么还不能直接发布

代码已推进到“上架准备态”，但仍不能跳过微信平台人工步骤：

- 当前 `project.config.json` 仍使用 `touristappid`。
- 需要用户在微信公众平台确认真实主体、备案、服务类目和隐私保护指引。
- 需要在微信开发者工具中完成真实编译、真机预览、体验版上传和审核提交。
- 部分学校字段仍为“待核实”，后续应持续补充学校官网详情页、住宿状态和校区信息。

## 用户接下来要做什么

1. 用微信开发者工具导入 `/Users/tom/Dev/suzhou_highschool_miniprogram`。
2. 本地使用 `touristappid` 编译预览。
3. 确认页面、筛选、收藏、目标记录和清除本地数据功能。
4. 替换真实 AppID。
5. 在微信公众平台完成备案、服务类目和隐私保护指引。
6. 上传体验版，真机复测。
7. 填写版本说明和审核备注，提交审核。

## 微信开发者工具验证步骤

1. 打开微信开发者工具。
2. 导入项目目录。
3. 确认 `project.config.json` 当前为 `touristappid` 或替换真实 AppID。
4. 点击编译。
5. 检查首页、学校库、学校详情、收藏、目标记录、数据说明、隐私说明、我的。
6. 执行真机预览。

## 上传体验版步骤

1. 替换真实 AppID。
2. 确认代码中不包含 AppSecret、API Key、Token、Cookie、账号密码。
3. 微信开发者工具点击上传。
4. 填写版本号和版本说明。
5. 在微信公众平台设置体验版。
6. 使用体验成员账号复测。

## 提交审核步骤

1. 确认服务类目与小程序功能匹配。
2. 填写隐私保护指引。
3. 使用 `docs/mp2_review_materials.md` 中的简介、服务内容说明和审核备注。
4. 提交审核。
5. 审核通过后再发布。
