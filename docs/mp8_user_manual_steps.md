# MP8 用户手动操作步骤

本文档用于完成 Codex 无法代替用户完成的微信开发者工具图形界面操作。Codex 已安装并尝试打开微信开发者工具，但不会填写真实 AppID、不会上传开发版本、不会提交审核、不会发布线上版本。

## 1. 打开微信开发者工具

本机已通过 Homebrew 安装：

- 应用路径：`/Applications/wechatwebdevtools.app`
- 可在 Finder 的“应用程序”中打开 `wechatwebdevtools.app`
- 也可以在终端执行：`open -a wechatwebdevtools`

如果后续应用不存在，可重新安装：

```bash
brew install --cask wechatwebdevtools
```

如果 Homebrew 不存在，先由用户手动安装 Homebrew，或从微信官方开发者工具页面下载安装微信开发者工具。不要使用第三方下载站。

## 2. 导入项目

打开微信开发者工具后选择：

1. 小程序
2. 导入项目
3. 项目目录填写：`/Users/tom/Dev/suzhou_highschool_miniprogram`
4. AppID 填写你自己的真实小程序 AppID
5. 后端服务选择：不使用云服务
6. 不要把真实 AppID、AppSecret、token、cookie 或其他密钥写入仓库文件

当前 `project.config.json` 仍保持 `appid: "touristappid"`，这是本地占位值。真实上传前只在微信开发者工具界面填写真实 AppID。

## 3. 编译检查

导入项目后点击“编译”。编译成功后逐页检查：

- 首页：是否正常展示项目说明、版本和基础入口
- 学校库：学校列表、搜索、筛选是否可用
- 学校详情：学校基础信息、官方来源说明、历史分数线展示是否正常
- 收藏：本地收藏添加和取消是否正常
- 学习目标：本地目标记录新增、查看、清除是否正常
- 数据说明：数据来源、版本和免责声明是否正常
- 隐私说明：不登录、不定位、不上传用户数据的说明是否正常
- 我的：版本号 `1.4.0` 和 releaseStatus 是否正常显示

如果 Console 出现红色报错，请复制完整红色报错全文，不要只截图最后一行。

## 4. 手机预览

1. 确认微信开发者工具已登录你的微信开发者账号。
2. 点击“预览”。
3. 使用绑定为开发者或体验成员的微信扫码。
4. 在手机上检查首页、学校库、学校详情、收藏、学习目标、数据说明、隐私说明和我的页面。
5. 如果手机端页面异常，记录机型、系统版本、微信版本、页面路径和完整 Console 红色报错。

## 5. 上传开发版本

确认编译和手机预览通过后，点击“上传”。

- 版本号：`1.4.0`
- 版本描述建议：

```text
MP6 最终收口版：展示苏州高中公开基础信息、2025 官方历史分数线、本地收藏和学习目标记录。
```

上传完成后，登录微信公众平台，在版本管理中确认开发版本已出现。

## 6. 设为体验版和添加体验成员

1. 进入微信公众平台。
2. 打开小程序版本管理。
3. 找到刚上传的开发版本。
4. 设置为体验版。
5. 在成员管理或体验成员管理中添加需要试用的微信号。
6. 让体验成员扫码或从体验入口打开，确认主要页面可用。

## 7. 提交审核

提交审核前确认：

- 页面不要求登录
- 不使用定位
- 不使用支付
- 不使用广告
- 不使用云服务
- 不上传用户个人数据
- 不提供录取预测或志愿填报结论
- 学校信息和历史分数线仅作为公开信息查询与学习目标记录参考

审核备注可沿用 MP7 文档中的说明，重点说明本小程序展示公开基础信息和历史分数线，收藏与学习目标仅保存在本地。

## 8. CLI 服务端口

Codex 已检测到微信开发者工具内置 CLI：

```text
/Applications/wechatwebdevtools.app/Contents/MacOS/cli
```

但 CLI 打开项目时提示服务端口关闭。若用户希望以后用 CLI 自动打开项目，可在微信开发者工具中手动设置：

1. 打开微信开发者工具
2. 进入“设置”
3. 进入“安全设置”
4. 打开“服务端口”

服务端口属于本机开发工具安全设置，本轮未由 Codex 自动开启。
