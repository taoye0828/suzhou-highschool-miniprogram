# MP1 严格审计修复报告

修复日期：2026-07-02

## 已修复

1. 本地存储统一捕获读取、写入、删除异常，不再向页面直接抛出。
2. 依赖旧数据的写操作会在读取失败时中止，避免用空数组覆盖已有收藏或目标记录。
3. 目标记录增加结构校验、旧数据归一化、版本字段和 100 条上限。
4. 页面只在存储操作真实成功后显示成功提示，并对读取失败给出一次性提示。
5. 目标草稿改为 400ms 防抖，并在页面隐藏或卸载时尝试刷新未保存草稿。
6. 目标 ID 增加时间戳、进程内序号和随机后缀。
7. 目标记录只持久化原始字段，差距、提醒和展示时间在页面读取时计算。
8. 自由备注增加敏感信息警告，隐私说明改为“不主动要求”，避免绝对声明与自由输入冲突。
9. 版本、分数边界、记录上限、数据版本和合规文案集中到 `config/app-config.js`。
10. 公开来源学校样本增加数据版本和来源核验日期。
11. 静态验证增加页面 JS 加载、WXML 事件处理器、全部项目文本密钥形态和环境文件检查。
12. 静态验证结果明确标记为补充检查，并明确开发者工具编译未执行。

## 故障注入覆盖

- 损坏目标记录和错误日期字段。
- 超过 100 条目标记录。
- 本地存储读取异常。
- 本地存储容量写入异常。
- 本地存储删除异常。
- 非法收藏 ID 和异常草稿结构。
